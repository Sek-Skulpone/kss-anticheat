# ระบบสอบออนไลน์ปลอดภัย โรงเรียนโคกสีวิทยาสรรค์ สพม.สกลนคร
### (KSS Anti-Cheat Online Exam WebApp)

ระบบเว็บแอปพลิเคชันสอบออนไลน์ที่ปลอดภัย มีระบบป้องกันและควบคุมการทุจริตอย่างเข้มงวด ออกแบบให้เป็น Serverless WebApp (Client-Side SPA) ที่สามารถนำขึ้นแสดงบน **GitHub Pages** ได้ทันทีโดยไม่มีค่าใช้จ่าย และเก็บฐานข้อมูลลงบน **Firebase Firestore** แบบประมวลผลทันที (Real-time)

---

## 🌟 ฟังก์ชันการทำงานเด่น
1. **ระบบความปลอดภัยและป้องกันการทุจริตสูงสุด (Anti-Cheat Engine)**:
   - บังคับสอบแบบเต็มจอ (Fullscreen)
   - ป้องกันและจำกัดการเข้าถึงบราวเซอร์: ป้องกันคลิกขวา, คลุมดำ, ก็อปปี้, วาง, พิมพ์หน้าจอ
   - ตรวจจับความถี่ในการสลับหน้าจอ สลับแท็บ หรือเปิดโปรแกรมอื่นๆ ทันที โดยมีโอกาสทำผิด 3 ครั้ง
   - เมื่อทำผิดกฎครบ 3 ครั้ง หน้าจอจะถูกล็อกสอบ (Locked) ทันที ไม่กดย้อนกลับหรือรีเฟรชได้
2. **ระบบปลดล็อกคุมสอบระยะไกลเรียลไทม์ (Live Exam Unlock)**:
   - คุณครูสามารถเปิดดูประวัติการทำผิดกฎของเด็กแต่ละคนแบบสดๆ ผ่านแผงควบคุมครู
   - คุณครูคนใดก็ได้สามารถสั่ง "ปลดล็อก" ให้เด็กที่ถูกล็อกหน้าจอสอบ โดยหน้าจอนักเรียนจะเด้งกลับเข้าสู่ข้อสอบทันทีโดยไม่ต้องทำการรีเฟรชหน้าเว็บ
3. **ระบบสลับข้อและตัวเลือกอัตโนมัติ (Exam Shuffling)**:
   - สุ่มเรียงคำถามและสุ่มเรียงลำดับตัวเลือก (ก, ข, ค, ง) ในการสอบของเด็กแต่ละคนโดยไม่ซ้ำกัน
   - ระบบเก็บข้อมูลความถูกต้องโดยอ้างอิงรหัส ID ของคำตอบ ทำให้การประเมินผลคะแนนแม่นยำ 100% แม้จะถูกสลับตำแหน่ง
4. **ตารางสอบจำลองเหมือนจริง**:
   - นักเรียนเข้าสู่ระบบเห็นตารางสอบเฉพาะระดับชั้นตัวเองเป็นแบบรายวัน (เช่น พุธ พฤหัสบดี ศุกร์ ตามภาพตารางหลักโรงเรียน)
   - ระบบนับเวลาถอยหลัง (Timer ticker) ก่อนเปิดข้อสอบให้เข้าทำ และนับเวลาทำข้อสอบเมื่อเริ่มทำจริง
5. **ตัวสแกนนำเข้ารายชื่อนักเรียนจาก Excel (SheetJS)**:
   - นำเข้าไฟล์รายชื่อจัดกลุ่มมัธยมแยกรายห้องแนวตั้งได้ในไฟล์เดียว (ตามตัวอย่าง `ใบรายชื่อนักเรียน.xls`)
6. **ตัวแปลงไฟล์ข้อสอบเวิร์ดเข้าสู่เว็บบนเบราว์เซอร์ (Docx parser)**:
   - อัปโหลดไฟล์ `.docx` ระบบจะแปลงคำถามและตรวจจับตัวเลือกที่ **ตัวหนา (Bold)** หรือ **ตัวขีดเส้นใต้ (Underline)** เพื่อใช้เป็นเฉลยอัตโนมัติ พร้อมหน้าจอให้ครูแก้ไขทวนทานได้

## 🛠️ วิธีการติดตั้งฐานข้อมูล (Firebase Firestore)

ระบบออกแบบให้ใช้งานคีย์การเชื่อมต่อแบบบันทึกในเบราว์เซอร์ส่วนตัว (Zero-Config Repo) คุณสามารถนำโค้ดไปอัปโหลดขึ้น GitHub ของคุณได้ทันทีโดยไม่มีคีย์รั่วไหล และตั้งค่าเชื่อมต่อหน้าแรกด้วยตนเอง:

1. สมัครใช้งานและล็อกอินเข้าสู่ [Firebase Console](https://console.firebase.google.com/)
2. กดสร้างโปรเจกต์ใหม่ (เช่น `kss-exam-system`)
3. ในแถบเมนูข้างๆ เลือก **Firestore Database** แล้วกด **Create Database**
   * เลือกเป็น **Start in production mode**
   * เลือกโซนที่ตั้งเซิร์ฟเวอร์ (เช่น `asia-southeast1` สิงคโปร์ หรือ `asia-east1` เพื่อความเร็วสูงสุดในไทย)
4. ไปที่ **Rules** ของ Firestore Database และแก้ไขสิทธิ์เป็นดังนี้ เพื่ออนุญาตให้ระบบทดสอบอ่านเขียนข้อมูลได้ (สามารถปรับสิทธิ์ความปลอดภัยเพิ่มได้ภายหลัง):
   ```javascript
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /{document=**} {
         allow read, write: if true;
       }
     }
   }
   ```
   *แล้วกด **Publish***
5. กลับไปที่หน้าโปรเจกต์ตั้งค่าระบบ (Project Settings) ด้านบนซ้าย
6. ไปด้านล่างตรงหัวข้อ **Your apps** เลือกรูป `</>` (Web App) เพื่อลงทะเบียนแอป
7. คัดลอกเฉพาะข้อมูลในวงเล็บปีกกาของ `firebaseConfig`:
   ```javascript
   const firebaseConfig = {
     apiKey: "AIzaSy...",
     authDomain: "...",
     projectId: "...",
     storageBucket: "...",
     messagingSenderId: "...",
     appId: "..."
   };
   ```
8. นำข้อมูลเหล่านี้ไปวางในหน้าจอเปิดใช้งานครั้งแรกสุดของเว็บบอร์ดระบบสอบ เพื่อทำการเชื่อมต่อกับฐานข้อมูลของโรงเรียน

## 🚀 วิธีอัปโหลดระบบขึ้น GitHub Pages (โฮสต์ฟรี)

1. สมัครใช้งานบัญชี [GitHub](https://github.com/) และทำการสร้างคลังเก็บข้อมูลใหม่ (New Repository) ตั้งชื่อว่า `kss-anticheat`
2. อัปโหลดไฟล์ต้นฉบับทั้งหมดจากโฟลเดอร์นี้ขึ้นไปบนคลังเก็บข้อมูล ได้แก่:
   - `index.html`
   - `index.css`
   - `app.js`
   - `firebase-db.js`
   - `xlsx-parser.js`
   - `docx-parser.js`
   - `README.md`
   - `ใบรายชื่อนักเรียน.xls`
3. ไปที่แถบเมนูการตั้งค่า **Settings** ของ Repository บนหน้าเว็บ GitHub
4. เลือกหัวข้อ **Pages** ทางด้านซ้ายมือ
5. ตรงหัวข้อ **Build and deployment** -> **Branch**
   - เปลี่ยนจาก `None` เป็น `main` (หรือ `master`)
   - เลือกเป็น `/ (root)` แล้วกด **Save**
6. รอประมาณ 1-2 นาที คุณจะได้ลิงก์หน้าเว็บไซต์สำหรับใช้งานระบบสอบออนไลน์ปลายภาคของโรงเรียน (เช่น `https://username.github.io/kss-anticheat/`)
